# 需求规约模板：RPA 网页账单与财务对账单自动导出规约 (RPA Invoice/Ledger Auto-Export)

## 1. 需求背景 (Background)
发包方在各大电商平台（如拼多多商家后台、淘宝卖家中心）拥有多个店铺，每日需导出繁琐的销售账单与对账明细文件。由于平台未提供开放 API 或 API 导出额度受限，发包方需要搭建一套 RPA（影刀、UiPath 或 Playwright/Puppeteer 脚本）自动化流程，模拟人工登录后台、进入财务中心、筛选日期并下载 `.xlsx`/`.csv` 账单到本地，实现财务流程的自动化。

---

## 2. 非目标 / 不承接范围 (Out of Scope)
*   **平台安全防御绕过**：不开发任何专门用于破解/绕过目标平台反爬虫、安全验证码（如滑动验证码、短信接码）的技术模块。
*   **黑客或暴力撞库**：不提供任何针对他人店铺的爬取或未授权的后台登录技术支持。
*   **自动提现与资金划转**：为确保资金绝对安全，本自动化流程仅限执行账单“只读导出”，任何涉及资金划扣、提现、退款等写入操作均不予承接。

---

## 3. 输入与输出规范 (Inputs & Outputs)
*   **输入端数据源**：
    *   发包方指定的商户后台登录凭证（由发包方在运行时通过受信任的安全环境输入，服务商不予保存）。
    *   查询控制参数：筛选时间区间（如“昨日起止时间”）、店铺标识列表。
*   **输出端目标**：
    *   自动下载的原始 Excel 账单文件，存储于发包方指定的本地目录或企业私有对象存储（如阿里云 OSS，包含标准的加密策略）。
    *   对账格式化输出：将下载的文件自动解析，规范输出为包含 `订单号`、`应收金额`、`实收金额`、`佣金比例` 的财务清洗表。

---

## 4. 验收标准 (Acceptance Criteria)
*   **操作流完整性**：在网络正常及后台界面结构未发生大改的前提下，RPA 能够模拟人工顺利走通“登录-筛选-点击下载-文件落盘”的完整链路。
*   **异常退出与状态通知**：若在运行过程中遇到人机挑战（CAPTCHA 验证阻断）或网络断开，系统应立即停止执行，记录当前的页面截图并触发管理员报警通知，而非盲目重试导致账号受限。
*   **维护保修期约定**：明确由于电商平台网页结构改版导致的 RPA 节点定位失效，在交付验收通过后 30 天内提供有限次数的免费适配，超出保修期则按人天维护标准收费。

---

## 5. 风险边界与费用约定 (Risk Boundaries & Cost Agreements)
*   **安全合规责任归属**：发包方必须拥有所操作商户店铺的合法所有权或完整授权。若因平台检测到异常登录频率导致店铺账号被降权、限流或封禁，相关风险完全由发包方自行评估并承担，服务商对此免责。
*   **运行代理及网络开销**：运行该 RPA 所需 of 独立安全浏览器环境、干净的代理 IP 节点及打码识别第三方服务费均由发包方直接采购，且仅能在平台 ToS 允许的合规授权场景下配置使用。

---

## 6. 可交付物列表 (Deliverables)
*   `[ ]` 影刀/UiPath RPA 工程包（或可执行 Python Playwright 源码）
*   `[ ]` RPA 环境变量与 XPath 定位节点配置文件
*   `[ ]` 本地异常页面截图与报警配置模块
*   `[ ]` 项目交付验收单及 30 天维护期约定协议

---

## 7. 适合 AI Agent 执行的任务切片 (Agent Task Slices)
1.  **静态 XPath 生成与校验**：解析商户后台财务页面的 HTML 结构，自动提取高鲁棒性的 DOM 元素 XPath/CSS 定位选择器。
2.  **账单数据清洗脚本**：自动编写清洗 Python 脚本，将多份格式不一的原始平台财务表格统一合并为结构化的 Pandas DataFrame。
3.  **Webhook 报警器封装**：编写通用钉钉/企业微信 Webhook 报警组件，用于在 RPA 遇到滑动验证码阻断时，发送截图给运营负责人处理。
