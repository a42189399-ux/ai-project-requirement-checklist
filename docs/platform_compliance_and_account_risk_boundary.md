# 平台合规性与账号风控边界指南 (Platform Compliance & Account Risk Boundaries)

> 💡 **写在前面：**  
> 很多 AI 落地、数据抓取和自动化运维项目，往往由于偏离了相关宿主平台的《服务条款（Terms of Service, ToS）》，导致项目在运行过程中出现“账号受限、封锁风控、接口欠费、数据确权”等非技术纠纷。  
> 本指南作为规约附件，旨在为发包方与承接方明确规定项目的合规底线与技术实施责任，防范合规红线造成的项目烂尾和资产损失。

---

## 1. 明确不予承接的非合规需求 (Non-Negotiable Restrictions)

为规避法律红线及账号封禁纠纷，本技术规约及后续配套模板包严禁承接或包含以下任何方向的技术方案：
1.  **平台封禁强行绕过**：不承接关于如何破解、绕过宿主官方平台的安全限制、IP黑名单屏蔽、反爬虫防火墙等对抗性技术。
2.  **人机验证码自动绕过**：不承接以撞库、暴力破解、或使用非法逆向 API 绕过第三方图形/短信/滑动人机挑战（CAPTCHA）的技术实现。
3.  **违规垃圾信息分发 (Spam)**：不承接以“自动洗稿”、“批量伪原创”、“垃圾群发引流”为目的的黑产营销工具开发。
4.  **违反平台 ToS 的非授权自动化**：不承接未获得数据所有者明示许可，以强行模拟协议等手段非法侵入他人私有数据接口的灰色软件开发。

---

## 2. 账号所有权与授权合规责任 (Account Ownership & Authorization)

在任何自动化系统的开发与部署中，账号管理遵循以下基本分工界限：
*   **主体凭证自持**：系统运行所需的大模型 API Key（如 OpenAI, DeepSeek）、云服务器账号、电商商户后台账号（如 Amazon, Shopify）、第三方应用平台凭证（如飞书、GitHub、Telegram）均须由**发包方自行注册、实名拥有并配置授权**。
*   **授权合法性陈述**：发包方须保证对自动化操作的数据源及目标宿主系统拥有绝对的使用权或合法的第三方 API 调用授权。承接方在开发时默认仅对发包方提供的接口定义进行技术实现，不承担因发包方授权缺陷导致的侵权法律责任。

---

## 3. 运行费率与持续服务开销 (Running Rates & Third-Party Service Costs)

系统运行产生的各项软硬件及服务费用不包含在软件开发交付款项中，其归属边界如下：
*   **消耗性 API 费用**：大模型执行消耗的 Token 费、第三方通知服务代发费等，均由发包方账户余额直接扣除。开发方提供的系统应包含“额度熔断阀门”，当检测到当月/当日账单额度超过发包方设定的预算水位时，系统应自动暂停运行以防止欠费超额。
*   **网络代理与 IP 资源采购**：若自动化系统在合规范围内确需使用代理 IP 池、CDN 或科学上网网络拓扑以完成跨国数据同步，该部分 IP 代理池年费、服务器带宽费由**发包方直接采购并负担费用**。

---

## 4. 防封风控与维护责任划分 (Account Ban Disclaimer & Maintenance)

*   **风控免责条款**：在合规框架内，即使使用官方公开 API 进行低频自动化，宿主平台亦可能根据其特有的风控规则（如检测到短时间内 IP 变化、同一凭证高频并发请求）对发包方的账号进行限制、安全警告或封禁。发包方须在发包前对此账号损失风险有充分认知，承接方对此类因平台自身规则变动造成的经营损失不承担任何连带赔偿责任。
*   **网页改版与维护保修**：由于 RPA 依赖于浏览器界面节点渲染（XPath/CSS），第三方平台界面的微调改版可能直接造成自动化定位失效。合同双方须提前以技术协议形式明确约定“验收合格后的免费质保周期（建议 30-90 天）”及超出该周期后的“改版适配维护人天报价”。

---

## 5. 合规合同范本推荐条款 (Recommended Compliance Clause for Contracts)

建议发包方与承接方在签署开发合同（SOW）时，将以下合规边界条款作为补充：
> “本合同所约定开发之系统，其运行环境及第三方接口账户由发包方自行提供并负责维护。发包方保证其所提供之账户、数据源及操作行为符合国家法律法规及各相关宿主平台之服务条款。因系统运行行为触发之平台账号风控、限制或封禁，承接方免于承担因此产生之直接或间接法律与经济赔偿责任。若因第三方平台界面布局变更导致定位脚本失效，由双方按照本协议约定的质保服务及改版适配费率另行协商解决。”
